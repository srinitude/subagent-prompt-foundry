# Research and Clarification

Research is mandatory for this skill.

## Go here when
- the request names a domain with official documentation
- terminology, standards, or current practices matter
- the prompt needs better constraints
- clarifying questions would be sharper after research

## Do not stop here when
- the user already gave enough structure to draft the prompt
- research has already answered the material ambiguities

## Clarification rule
Ask questions only after research if material ambiguity remains.

Ask about:
- role boundaries
- output format
- allowed sources
- evaluation criteria
- tool assumptions
- parent↔subagent handoff

Avoid:
- "Can you provide more context?"
- redundant questions already answerable from research or supplied files

## Source priority
1. first-party
2. official docs
3. standards/specs
4. strong third-party
